Wildfire Evacuation Planning Methodology
FIRESCENE Project – Evacuation Module

1. Overview

The wildfire evacuation module of the FIRESCENE project provides a scenario-based, parameter-driven approach to estimate evacuation requirements, population dynamics, and associated resource needs for defined risk zones. This tool supports strategic emergency preparedness and response planning for wildfires in WUI areas, aligning with the broader goal of enhancing situational awareness, risk-informed contingency planning, and evacuation coordination under the UCPM framework.

It operationalizes a simplified but dynamic model to calculate population distribution, evacuee profiles, transportation timelines, and shelter requirements for wildfire-prone areas. The system supports real-time configuration of demographic and logistic parameters, offering decision-support outputs for local Civil Protection authorities.

2. Methodological Framework

The module is designed to answer critical operational planning questions:

  • How many people must be evacuated under a given scenario?
  • What are their demographic profiles and vulnerabilities?
  • How fast can they be mobilized to safety?
  • What is the capacity gap in terms of sheltering and transport?

The system assumes zone-based population segmentation and evacuation timelines based on preconfigured parameters. These parameters can be adjusted through the user interface.

3. Core Input Parameters and Definitions

Users provide inputs to simulate different evacuation scenarios. Parameters are grouped into the following dimensions:

3.1. Demographic Structure

Total Zone Population (P): Number of people within the evacuation area (can be user-entered).

Age Distribution:

  • Children (<12): %
  • Adults (13–60): %
  • Elderly (>60): %
  • (Sum must equal 100% – interface logic ensures dependency across categories)

Gender Distribution:

  • % Male
  • % Female
  • (Mutually linked to sum to 100%)

Vulnerable Groups:

  • % of population with limited mobility, disabilities, or cognitive impairments (e.g., elderly, tourists, medically fragile).

3.2. Population Behavior & Evacuation Logic

Parameter Description Units/Range
Compliance Rate Proportion of population likely to follow evacuation orders Percentage (%)
Self-Evacuation % of population with own means to evacuate Percentage (%)
Assisted Evacuation % requiring external transport (e.g., buses) Percentage (%)
Sheltering In Place % not evacuated but sheltering in defensible structures Percentage (%)
Avg Speed to Assembly Points Influences overall time-to-safety km/h
Avg Distance to Assembly Points Used with speed to estimate travel time km

3.3. Shelter and Transport Logistics

Parameter Description Application
Shelter Capacity Number of evacuees that can be hosted in formal shelters Within or outside the zone
Transport Units Required Based on evacuees without self-evacuation capacity Vehicle capacity assumptions (typically buses)

4. Calculation Methodology

Once the parameters are entered or adjusted, the system computes the following key metrics:

4.1. Effective Evacuee Population (E)

Formula:
E = P · (Compliance Rate) − (Sheltering in Place Population)

4.2. Evacuees Requiring Assistance (A)

Formula:
A = E · (Assisted Evacuation Share)

4.3. Time to Assembly (Ta)

Formula:
Ta = Average Distance to Assembly Point (km) / Average Speed (km/h)

4.4. Shelter Gap (SG)

Formula:
SG = max(0, E − Available Shelter Capacity)

This figure identifies whether additional shelter facilities are needed or if current infrastructure suffices.

4.5. Transport Vehicle Needs (TVN)

Formula:
TVN = ⌈A / Average Bus Capacity (default assumed = 50)⌉

5. System Outputs

The interface delivers:

  • A bar chart representing evacuee groups and their needs
  • Pop-up explanations for computed evacuation dynamics
  • Risk-informed decision support visualizations for planners
  • Downloadable summary report (if working correctly) for documentation

5.1. Classification and Visualization

  • Vulnerability Distribution: Shown as color-coded bars
  • Assembly Travel Time: Converted into total evacuation timeline estimates
  • Capacity Gaps: Flagged for contingency planning (e.g., shelter expansion or bus requisition)
  • Dynamic Results: Scenario-sensitive and change dynamically with slider input

6. Spatial Integration and GIS Applications

While the interface operates at the zone scale, the same logic can be extended spatially by:

GIS Extension Workflow:
  1. Mapping demographic distributions via census data
  2. Estimating travel time surfaces to assembly points using road network data
  3. Overlaying shelter locations and capacity constraints
  4. Generating zonal evacuation plans and time-constrained routing scenarios

This approach allows evacuation plans to be grounded in real topography, infrastructure, and mobility data.

8. Conclusion

The Firescene evacuation tool provides a transparent, modular, and user-configurable evacuation estimation framework tailored for wildfire-prone WUI areas. It translates critical demographic and behavioral factors into operational planning indicators, offering Civil Protection authorities a lightweight yet effective platform for proactive evacuation strategy design.

It complements the integrated wildfire risk index module by transitioning from risk understanding to response preparedness, supporting the full continuum of disaster risk management as promoted in the Firescene project.
